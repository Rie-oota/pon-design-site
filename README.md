# PON DESIGN Corporate Site

PON DESIGNのコーポレートサイトを想定した架空サイトです。
Figmaで作成されたデザインカンプをもとに、実務を意識した設計・実装で制作しました。

---

## 使用技術

- HTML
- SCSS
- JavaScript
- Nunjucks
- Gulp

---

## 技術選定

本制作では、複数ページで共通するヘッダー・フッターを効率的に管理するため、Nunjucksを使用しました。

静的HTMLのみで制作した場合、共通パーツを各ページに記述する必要があり、修正時に複数ファイルの更新が発生します。
Nunjucksを使用することで、共通パーツとして切り分け、保守性の高い構造を実現しています。

---

## 制作背景

Figmaで作成されたデザインカンプをもとにコーディングを行いました。
デザインの再現性だけでなく、レスポンシブ対応やコンポーネント設計を意識し、
実務で運用しやすい構造を目指しています。

---

## 実装内容

- レスポンシブ対応（PC / タブレット / スマホ）
- ヘッダー・フッターの共通化（Nunjucks）
- SCSSによるスタイル管理
- CSS変数によるデザイントークン管理
- お問い合わせフォーム（JSで入力内容をコンソール出力）

---

## 設計

CSS設計にはFLOCSSを採用し、以下のルールでクラス設計を行いました。

- layout（l-）：レイアウト構造
- component（c-）：再利用可能なUIパーツ
- project（p-）：ページ固有のスタイル

また、デザインと役割を分離するために、
ボタンなどは「共通クラス」と「ページ固有クラス」を組み合わせて実装しています。

---

## スタイル設計

- 共通レイアウト（ヘッダー・フッター・基本スタイル）はCSSで管理
- ページごとのスタイルはSCSSで記述し、style.cssに集約
- SCSSは保守性と再利用性を意識し、ページ単位で分割
- カラーはCSS変数（:root）で一元管理
- フォントサイズはremベースで設計

## 開発環境の立ち上げ方

※Node.jsがインストールされている前提です。

依存パッケージをインストールし、Gulpを起動します。

```bash
npm install
npx gulp
```

---

## ビルドについて

本プロジェクトでは、Nunjucksを用いてテンプレートをHTMLに変換しています。

編集対象：`src/*.njk`
出力先：`dist/*.html`

src内の.njkファイルはビルド時にHTMLへ変換され、distフォルダに出力されます。
ブラウザで確認する際は、dist内のHTMLファイルを開いてください。

distフォルダはビルドによって生成されるため、Git管理対象には含めていません。

---

## スタイルのビルドについて

SCSSはGulpを使用してコンパイルしています。
`npx gulp` 実行時に、`scss/style.scss` が `css/style.css` に変換されます。
